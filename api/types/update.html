class aiogram.types.update.Update(*, update_id: int, message: Message | None = None, edited_message: Message | None = None, channel_post: Message | None = None, edited_channel_post: Message | None = None, message_reaction: MessageReactionUpdated | None = None, message_reaction_count: MessageReactionCountUpdated | None = None, inline_query: InlineQuery | None = None, chosen_inline_result: ChosenInlineResult | None = None, callback_query: CallbackQuery | None = None, shipping_query: ShippingQuery | None = None, pre_checkout_query: PreCheckoutQuery | None = None, poll: Poll | None = None, poll_answer: PollAnswer | None = None, my_chat_member: ChatMemberUpdated | None = None, chat_member: ChatMemberUpdated | None = None, chat_join_request: ChatJoinRequest | None = None, chat_boost: ChatBoostUpdated | None = None, removed_chat_boost: ChatBoostRemoved | None = None, **extra_data: Any)[源代码]#

这个’对象’ <https:>_表示传入的更新。

在任何给定的更新中最多可以存在一个**可选参数。

来源： https://core.telegram.org/bots/api#update

update_id: int#

更新的唯一标识符。更新标识符从某个正数开始并顺序增加。如果您使用’webhooks <https:>’_，这个ID变得特别方便，因为它允许您忽略重复的更新或恢复正确的更新顺序，如果他们不按顺序。如果至少一周没有新的更新，则将随机地而不是顺序地选择下一更新的标识符。

message: Message | None#

可选。任何类型的新传入消息-文本，照片，贴纸等。

edited_message: Message | None#

可选。bot已知且已编辑的消息的新版本

channel_post: Message | None#

可选。任何类型的新传入频道帖子-文本，照片，贴纸等。

edited_channel_post: Message | None#

可选。bot已知并已编辑的频道帖子的新版本

message_reaction: MessageReactionUpdated | None#

可选。用户更改了对消息的反应。机器人必须是聊天中的管理员，并且必须在* allowed_updates *列表中明确指定： code:’“message_reaction "’才能接收这些更新。未收到由机器人设置的反应的更新。

message_reaction_count: MessageReactionCountUpdated | None#

可选。对具有匿名反应的消息的反应已更改。机器人必须是聊天中的管理员，并且必须在* allowed_updates *列表中明确指定： code:’“message_reaction_count "’才能接收这些更新。

inline_query: InlineQuery | None#

可选。新的传入’内联<https:>’_查询

chosen_inline_result: ChosenInlineResult | None#

可选。用户选择并发送到其聊天伙伴的“内联<https:>”查询的结果。有关如何为您的机器人启用这些更新的详细信息，请参阅我们的“反馈收集<https:>”文档。

callback_query: CallbackQuery | None#

可选。新的传入回调查询

shipping_query: ShippingQuery | None#

可选。新的进货运输查询。仅适用于价格灵活的发票

pre_checkout_query: PreCheckoutQuery | None#

可选。新传入的预结账查询。包含有关结帐的完整信息

poll: Poll | None#

可选。新的投票状态。机器人仅接收由机器人发送的关于已停止的投票和投票的更新

poll_answer: PollAnswer | None#

可选。用户在非匿名投票中更改了答案。机器人仅在机器人本身发送的民意测验中获得新的选票。

my_chat_member: ChatMemberUpdated | None#

可选。机器人的聊天成员状态已在聊天中更新。对于私人聊天，仅当机器人被用户阻止或取消阻止时才会收到此更新。

chat_member: ChatMemberUpdated | None#

可选。聊天成员的状态已在聊天中更新。机器人必须是聊天中的管理员，并且必须在* allowed_updates *列表中明确指定： code:’“chat_member "’才能接收这些更新。

chat_join_request: ChatJoinRequest | None#

可选。已发送加入聊天的请求。机器人必须在聊天中拥有* can_invite_users *管理员权限才能接收这些更新。

chat_boost: ChatBoostUpdated | None#

可选。添加或更改了聊天增强功能。机器人必须是聊天中的管理员才能接收这些更新。

removed_chat_boost: ChatBoostRemoved | None#

可选。从聊天中删除了boost。机器人必须是聊天中的管理员才能接收这些更新。

property event_type: str#

检测更新类型如果更新类型未知，则提升UpdateTypeLookupError

返回:

property event: TelegramObject#
exception aiogram.types.update.UpdateTypeLookupError[源代码]#

更新不包含任何已知的事件类型。
