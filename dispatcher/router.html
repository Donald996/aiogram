路由器(Router)#

用法：

from aiogram import Router
from aiogram.types import Message

my_router = Router(name=__name__)

@my_router.message()
async def message_handler(message: Message) -> Any:
    await message.answer('Hello from my router!')
class aiogram.dispatcher.router.Router(*, name: str | None = None)[源代码]#

基类：object

Router 可以路由更新，它嵌套更新类型，如消息，回调查询，轮询和所有其他事件类型。

事件处理程序可以通过两种方式在观察者中注册：

  • 通过观察者方法 - router.<event_type>.register(handler, <filters, …>)

  • 通过装饰器 - @router.<event_type>(<filters, …>)

参数:

name – 可选的 router 名称，可用于调试

连接另一个 router。

参数:

router –

返回:

include_routers(*routers: Router) → None[源代码]#

连接多个 router。

参数:

routers –

返回:

resolve_used_update_types(skip_events: Set[str] | None = None) → List[str][源代码]#

解析已注册的事件名称

对于仅获取已注册事件类型的更新非常有用。

参数:

skip_events – 跳过指定的事件名称

返回:

一组注册名称

事件观察者#

警告

所有处理程序都应该是异步的。处理程序函数的名称并不重要。事件参数名称也不重要，但建议不要与上下文数据重叠，因为函数不能接受两个同名参数。

以下是可用的观察者列表以及如何注册处理程序的示例

在这些示例中，仅使用装饰器风格的注册处理程序，但如果您不喜欢 @decorators(装饰器)，只需使用 <event type>.register(…) 方法即可。

嵌套 router#

警告

顺便说一下，Routers 可以嵌套到另一个 Routers，但有一些限制：

1.Router **不能**包括自身 1.Router **不能**用于循环包括（Router 1 包括Router 2，Router 2 包括Router3，Router 3 包括Router 1）

示例：

module_1.py#
name:

module_1

router2 = Router()

@router2.message() …

module_2.py#
name:

module_2

from module_2 import router2

router1 = Router() router1.include_router(router2)

更新#

@dispatcher.update()
async def message_handler(update: types.Update) -> Any: pass

警告

只有根 Router（Dispatcher）可以处理这类事件。

备注

Dispatcher 已经为该事件类型提供了默认处理程序，因此可以用它来处理其他处理程序未处理的所有更新。
