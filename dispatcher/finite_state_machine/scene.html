场景¶

Added in version 3.2.

警告

这个功能是实验性的，未来版本可能会发生变化。

aiogram 的基本API易于使用且功能强大，允许实现简单的交互，比如触发命令或消息以获取响应。然而，某些任务需要用户和机器人之间的对话。这就是场景发挥作用的地方。

理解场景¶

在 aiogram 中，一个场景就像是一个抽象的、孤立的命名空间或房间，用户可以通过代码引导进入其中。当用户处于场景中时，大多数其他全局命令或消息处理程序都会被绕过，除非它们被专门设计为在场景之外起作用。这有助于创建一种专注交互的体验。场景为更复杂的交互提供了结构，有效地隔离和管理对话不同阶段的上下文。它们允许您以更有组织的方式控制和管理对话的流程。

场景生命周期(Scene Lifecycle)¶

每个场景都可以被 entered ,`left` 或 exited ，从而清晰地过渡到对话的不同阶段。例如，在多步骤表单填写交互中，每个步骤都可以是一个场景-机器人引导用户从一个场景到下一个，当他们提供所需的信息。

场景监听器(Scene Listeners)¶

场景有自己的钩子，这些钩子是命令或消息监听器，只会在用户在场景内时起作用。这些钩子对用户的动作做出反应，而用户在 内部 场景时，提供适合该上下文的响应或动作。当用户从一个场景被引导到另一个场景时，行动和响应相应地改变，因为用户现在正在与新场景内的监听器集交互。这些 特定场景 的钩子或监听器，脱离了全局监听上下文，允许更加流畅和有组织的机器人用户交互。

场景互动(Scene Interactions)¶

每个场景就像是一个独立的世界，其内定义了场景范围内的互动。因此，在特定场景内定义的处理程序才会在该场景的生命周期内对用户的输入作出反应。

场景优势(Scene Benefits)¶

场景可以帮助管理更复杂的交互工作流程，使用户与机器人之间的对话更加互动和动态。这样可以更灵活地处理多步交互或与用户的对话。

如何使用场景¶

举例来说，我们有一个问答机器人，它会向用户提出一系列问题，然后展示结果。

让我们从数据模型开始，在这个例子中，简单的数据模型用于表示问题和答案，在现实生活中，您可能会使用数据库来存储数据。

class Question:

    text: str

    answers: list[Answer]
    """List of answers"""

    correct_answer: str = field(init=False)

    def __post_init__(self):
        self.correct_answer = next(answer.text for answer in self.answers if answer.is_correct)

class QuizScene(Scene, state="quiz"):

    """

还需要定义一个处理程序来帮助启动小测验游戏：

开始命令处理程序¶
quiz_router = Router(name=__name__)
# Add handler that initializes the scene
quiz_router.message.register(QuizScene.as_handler(), Command("quiz"))

一旦场景被定义，我们需要在SceneRegistry中注册它：

注册场景¶
def create_dispatcher():
    # Event isolation is needed to correctly handle fast user responses
    dispatcher = Dispatcher(
        events_isolation=SimpleEventIsolation(),
    )
    dispatcher.include_router(quiz_router)

    # To use scenes, you should create a SceneRegistry and register your scenes there
    scene_registry = SceneRegistry(dispatcher)
    # ... and then register a scene in the registry
    # by default, Scene will be mounted to the router that passed to the SceneRegistry,
    # but you can specify the router explicitly using the `router` argument
    scene_registry.add(QuizScene)

    return dispatcher

所以，现在我们可以实现问答游戏逻辑，每个问题会逐个发送给用户，用户的答案会在所有问题结束后检查。

现在我们需要为问题处理程序编写一个入口点：

问题处理程序入口¶
    @on.message.enter()
    async def on_enter(self, message: Message, state: FSMContext, step: int | None = 0) -> Any:
        """
        Method triggered when the user enters the quiz scene.

        It displays the current question and answer options to the user.

        :param message:
        :param state:
        :param step: Scene argument, can be passed to the scene using the wizard
        :return:
        """
        if not step:
            # This is the first step, so we should greet the user
            await message.answer("Welcome to the quiz!")

        try:
            quiz = QUESTIONS[step]
        except IndexError:
            # This error means that the question's list is over
            return await self.wizard.exit()

        markup = ReplyKeyboardBuilder()
        markup.add(*[KeyboardButton(text=answer.text) for answer in quiz.answers])

        if step > 0:
            markup.button(text="🔙 Back")
        markup.button(text="🚫 Exit")

        await state.update_data(step=step)
        return await message.answer(
            text=QUESTIONS[step].text,
            reply_markup=markup.adjust(2).as_markup(resize_keyboard=True),
        )

一旦进入场景，我们应该期望用户的回答，所以我们需要为它编写一个处理程序，这个处理程序应该期望文本消息，保存答案，并重新获取下一个问题的问题处理程序：

答案处理程序¶
    @on.message(F.text)
    async def answer(self, message: Message, state: FSMContext) -> None:
        """
        Method triggered when the user selects an answer.

        It stores the answer and proceeds to the next question.

        :param message:
        :param state:
        :return:
        """
        data = await state.get_data()
        step = data["step"]
        answers = data.get("answers", {})
        answers[step] = message.text
        await state.update_data(answers=answers)

        await self.wizard.retake(step=step + 1)

class QuizScene(Scene, state="quiz"):
    """
    This class represents a scene for a quiz game.

    It inherits from Scene class and is associated with the state "quiz".
    It handles the logic and flow of the quiz game.
    """

    @on.message.enter()
    async def on_enter(self, message: Message, state: FSMContext, step: int | None = 0) -> Any:
        """
        Method triggered when the user enters the quiz scene.

        It displays the current question and answer options to the user.

        :param message:
        :param state:
        :param step: Scene argument, can be passed to the scene using the wizard
        :return:
        """
        if not step:
            # This is the first step, so we should greet the user
            await message.answer("Welcome to the quiz!")

        try:
            quiz = QUESTIONS[step]
        except IndexError:
            # This error means that the question's list is over
            return await self.wizard.exit()

        markup = ReplyKeyboardBuilder()
        markup.add(*[KeyboardButton(text=answer.text) for answer in quiz.answers])

        if step > 0:
            markup.button(text="🔙 Back")
        markup.button(text="🚫 Exit")

        await state.update_data(step=step)
        return await message.answer(
            text=QUESTIONS[step].text,
            reply_markup=markup.adjust(2).as_markup(resize_keyboard=True),
        )

    @on.message.exit()
    async def on_exit(self, message: Message, state: FSMContext) -> None:
        """
        Method triggered when the user exits the quiz scene.

        It calculates the user's answers, displays the summary, and clears the stored answers.

        :param message:
        :param state:
        :return:
        """
        data = await state.get_data()
        answers = data.get("answers", {})

        correct = 0
        incorrect = 0
        user_answers = []
        for step, quiz in enumerate(QUESTIONS):
            answer = answers.get(step)
            is_correct = answer == quiz.correct_answer
            if is_correct:
                correct += 1
                icon = "✅"
            else:
                incorrect += 1
                icon = "❌"
            if answer is None:
                answer = "no answer"
            user_answers.append(f"{quiz.text} ({icon} {html.quote(answer)})")

        content = as_list(
            as_section(
                Bold("Your answers:"),
                as_numbered_list(*user_answers),
            ),
            "",
            as_section(
                Bold("Summary:"),
                as_list(
                    as_key_value("Correct", correct),
                    as_key_value("Incorrect", incorrect),
                ),
            ),
        )

        await message.answer(**content.as_kwargs(), reply_markup=ReplyKeyboardRemove())
        await state.set_data({})

    @on.message(F.text == "🔙 Back")
    async def back(self, message: Message, state: FSMContext) -> None:

def create_dispatcher():
    # Event isolation is needed to correctly handle fast user responses
    dispatcher = Dispatcher(
        events_isolation=SimpleEventIsolation(),
    )
    dispatcher.include_router(quiz_router)

    # To use scenes, you should create a SceneRegistry and register your scenes there
    scene_registry = SceneRegistry(dispatcher)
    # ... and then register a scene in the registry
    # by default, Scene will be mounted to the router that passed to the SceneRegistry,
    # but you can specify the router explicitly using the `router` argument
    scene_registry.add(QuizScene)

    return dispatcher

标记¶

标记与路由器事件注册机制类似，但用于标记场景类中的场景处理程序。

它可以从 from aiogram.fsm.scene import on 导入，并且应该用作装饰器。

允许的事件类型：

  • message

  • edited_message

  • channel_post

  • edited_channel_post

  • inline_query

  • chosen_inline_result

  • callback_query

  • shipping_query

  • pre_checkout_query

  • poll

  • poll_answer

  • my_chat_member

  • chat_member

  • chat_join_request

每种事件类型都可以像在路由器中一样进行过滤。

同样，每个事件类型都可以标记为场景入口点、出口点或离开点。

如果你想标记可以从消息或内联查询进入的场景，你应该使用 on.message 或 on.inline_query 标记：

class MyScene(Scene, name="my_scene"):
    @on.message.enter()
    async def on_enter(self, message: types.Message):
        pass

    @on.callback_query.enter()
    async def on_enter(self, callback_query: types.CallbackQuery):
        pass

Scene has only three points for transitions:

  • 进入点 - 当用户进入场景时

  • 离开点 - 当用户离开场景并进入另一个场景

  • 退出点 - 当用户从场景中退出
